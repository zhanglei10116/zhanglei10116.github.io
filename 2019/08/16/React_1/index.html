React 组件生命周期

React组件的生命周期分为三大部分：

  • 挂载 (Mount) ：组件第一次在DOM树中渲染
  • 更新 (Update) ：组件更新，即重渲染
  • 卸载 (Unmount)：把组件从DOM树中删除

1.挂载(Mount)

定义：把组件渲染，并且构造DOM元素插入到页面的过程称为组件的挂载
组件挂载的过程（组件挂载先关函数，只在组件挂载的时候调用，在组件更新时不会执行）

  • constructor()：组件初始化，初始化state值。
  • componmentWillMount()：组件挂载之前执行，在render函数之前执行，仅在挂载的时候执行一次，即使后续组件被多次重渲染，也不会再执行。
  • render()：React渲染。
  • componentDidMount()：组件挂载后执行，在render函数之后执行，仅在挂载的时候执行一次，即使后续组件被多次重渲染，也不会再执行。

2.更新(Update)

组件更新相关函数，在组件初始挂载的时候都不会调用。

  • componentWillReceiveProps(nextProps)
    当父组件传递给子组件的props发生变化时调用 ，该函数有一个参数nextProps（即新的Props的值），this.props可以获取到当前的props（即旧的props），通过对比nextProps与this.props来重渲染组件。
  • shouldComponentUpdate(nextProps,nextState)
    在组件接收到新的props或state时调用，通过返回值来确定是否重新渲染当前组件，函数的两个参数，nextProps和nextState，分别表示获取到的新的props和新的state；
  • componentWillUpdate(nextProps,nextState)
    在shouldComponentUpdate执行并返回了true的时候被调用，组件开始重渲染，该函数的两个参数，nextProps和nextState，分别表示获取到的新的props和新的state；
  • componentDidUpdate(prevProps,prevState)
    组件更新，重渲染完毕后调用，该函数也有两个参数，prevProps 和prevState ，分别表示组件更新前的props和state。

3.卸载(Unmount)

componentWillUnmount函数在组件卸载的时候执行。
